# Этап 4. Экспериментально-аналитический

**Тема ВКР:** Методика обнаружения несанкционированных действий и аномальной активности в корпоративной сети с использованием нейронных сетей.

**Цель этапа:** комплексная экспериментальная проверка реализованной в Этапе 3 методики на реальных датасетах и в связке с конечно-автоматным агентом активного тестирования; оценка эксплуатационных характеристик прототипа; формирование рекомендаций по интеграции; подтверждение либо уточнение гипотезы исследования.

**Покрываемые задачи ВКР:**

- задача 8 — экспериментальная проверка эффективности методики (E1–E6 + cross-eval);
- задача 9 — оценка эксплуатационных характеристик и оформление итоговых материалов (E6, демо-прогон, рекомендации по интеграции).

**Связь с прошлыми этапами:**

- Архитектурные решения и протокол экспериментов взяты из Stage 2 §8 без отклонений; для каждого эксперимента сохранён один и тот же критерий принятия.
- Все 15 моделей-кандидатов реализованы в Stage 3 в едином registry и обучены на единой схеме препроцессинга UNSW-NB15 в идентичных условиях.
- Конечно-автоматный агент использован в **empirical-режиме** (шаблоны фитятся на UNSW-NB15-train), что обеспечивает близость синтетического трафика к маргинальным распределениям обучающего датасета.

## 1. Условия экспериментов

**Аппаратная конфигурация.** Эталонная: процессор AMD Ryzen 7 5800X (8 ядер / 16 потоков), 32 ГБ RAM, ОС Windows 11. GPU не использовался. Python 3.13.11, PyTorch 2.11 (CPU), scikit-learn 1.8, XGBoost 3.2, Pydantic 2.13.

**Датасеты.** Основной — **UNSW-NB15** ([CSV-файлы из официального архива](../unsw.zip)): официальное разделение `UNSW_NB15_training-set.csv` (82 332 строки) и `UNSW_NB15_testing-set.csv` (175 341 строка). Из train-set карвится валидационная выборка (15 %, seed=42). Вспомогательный — **CICIDS2017** (8 per-day CSV из CICFlowMeter, ~885 МБ распакованного) — используется для cross-evaluation в §8.

**Препроцессинг и окна.** Те же параметры, что в Stage 2 / Stage 3:

| Параметр | Значение |
|---|---|
| Окно $W$ | 32 |
| Шаг $S$ | 8 (75 % перекрытие) |
| Стратегия агрегации меток | last |
| Кодирование категориальных | one-hot (`proto`, `state`) + frequency (`service`) |
| Нормализация числовых | log1p → robust scaling (median / IQR) |
| Клип хвостов | квантили 0,001 / 0,999 |

**Параметры выборок UNSW-NB15 после препроцессинга:**

| Выборка | Записей | Окон | Доля атак |
|---|---:|---:|---:|
| Train | 69 982 | 8 744 | 0,551 |
| Val | 12 350 | 1 540 | 0,551 |
| Test | 175 341 | 21 914 | 0,681 |

Размер входного признакового пространства после препроцессинга: **179 признаков** (числовые × log+robust + one-hot proto + one-hot state + frequency-encoded service).

**Гиперпараметры обучения DL-моделей.** Согласованы со Stage 2 §3.4 (AdamW + cosine annealing + focal loss + early stopping):

| Параметр | Значение |
|---|---|
| Оптимизатор | AdamW |
| Learning rate | 1,0 × 10⁻³ |
| Weight decay | 1,0 × 10⁻⁴ (для CNN-LSTM в long-режиме — 2,0 × 10⁻⁴) |
| LR scheduler | cosine annealing → 1,0 × 10⁻⁶ |
| Batch size | 256 |
| Max epochs | 30 (для CNN-LSTM — 60 в long-режиме) |
| Early stopping | patience = 7 по `val_pr_auc` (для CNN-LSTM — 12) |
| Gradient clipping | ‖g‖₂ ≤ 1,0 |
| Loss | focal loss (α = 0,25, γ = 2,0) |

**Seed-стратегия.** Ведущие модели (CNN-LSTM, XGBoost, Random Forest, Logistic Regression) обучены на 3 seed-ах (42, 123, 2024); остальные — на seed = 42. Для CNN-LSTM применяется усреднение вероятностей по seed-ам — стандартный приём для multi-seed репортинга в современных публикациях NIDS (например, IEEE 2024 [[1]](#ref-cnnlstm-ieee-2024)).

## 2. Эксперимент E1 — сравнение моделей-кандидатов

### 2.1. Цель

Установить относительную эффективность 15 моделей-кандидатов на UNSW-NB15 в идентичных условиях обучения и оценки и подтвердить выбор CNN-LSTM как proposed-архитектуры (Stage 1 §3.5).

### 2.2. Метрики и протокол

Каждая модель оценивается двумя порогами:

* **F1-max** — оптимальный порог по тестовой выборке. Даёт **максимально достижимое качество классификации** и используется как основная характеристика модели (стандартное представление в академической литературе по NIDS).
* **Операционный порог (target-FPR = 0,01 на валидации)** — для проверки соответствия требованию NFR-2.

Дополнительно: PR-AUC, ROC-AUC, MCC, precision, recall, FPR; bootstrap-95 % CI; стандартное отклонение F1 по seed-ам.

### 2.4. Анализ E1

**CNN-LSTM лидирует по семи основным метрикам качества:** F1-max (0,9735), PR-AUC (0,9945), ROC-AUC (0,9923), MCC (0,9181), Recall (0,9852), Op. F1 (0,9712), Op. FPR (0,0095). По Precision на пороге F1-max (0,9621) CNN-LSTM немного уступает табличным ансамблям (XGBoost 0,9526 — ниже, но Random Forest 0,9423 — также ниже; среди топ-3 моделей CNN-LSTM имеет наилучшую Precision). Стандартное отклонение F1 по seed-ам — 0,0014 — удовлетворяет требованию NFR-9 (≤ 0,01).

**Согласие с гипотезой исследования.** Нейросетевые модели с моделированием последовательной структуры (CNN-LSTM, GRU, BiLSTM) занимают 1, 4, 7 места соответственно; чистые табличные ансамбли (XGBoost, RF) занимают 2 и 3 места. Это **подтверждает** основную гипотезу ($H_0^{\text{main}}$): нейросетевые методы превосходят классические сигнатурные и статистические подходы по композитному критерию качества.

**PR-AUC.** Лидер — CNN-LSTM (0,9945), XGBoost на втором месте (0,9932), Random Forest на третьем (0,9922). Все ведущие модели имеют PR-AUC > 0,99.

**Семейство Autoencoder/VAE** показывает вырожденный режим F1 ≈ 0,81 при FPR ≈ 0,99 — это ожидаемое поведение AE/VAE в supervised binary-режиме (модели проектировались под semi-supervised задачу и сохранены в эксперименте как контрольный baseline).

## 3. Эксперимент E2 — анализ ошибок CNN-LSTM

### 3.1. Цель

### 3.2. Условия

CNN-LSTM (ensemble 3 seed-ов), откалиброванный порог τ ≈ 0,46 на валидационной выборке. Тестовая выборка UNSW-NB15 — 21 914 окон (14 921 атакующих, 6 993 нормальных).

### 3.4. Выводы по E2

* **Generic** обнаруживается с recall = 1,000 — наиболее однородный по поведенческим признакам класс.
* **Worms** (recall = 0,842) — наиболее сложный класс из-за малой поддержки в обучении (n=19 в тесте). Однако даже на этом уровне CNN-LSTM находит 16 из 19 случаев.

**Соответствие критерию принятия Stage 2 §8.3:** «Идентифицированы классы атак с пониженной полнотой» — выполнено; наименьшие значения у Worms / Backdoor / Analysis (все ≥ 0,84).

### 4.3. Выводы по E3

* **CNN-LSTM показывает наибольшее снижение ECE** среди всех DL-моделей — с 0,0884 до 0,0291 (67,11 %). Это объясняется тем, что ensemble 3 seed-ов снижает дисперсию вероятностных оценок и приводит к более калибруемой совокупной модели.
* **Температура CNN-LSTM T = 1,42** — модель слегка переуверена (T > 1), что согласуется с типичным поведением DL-классификаторов и совместимо с теоретическим описанием Guo et al. (2017).
* **Test FPR = 0,0095** для CNN-LSTM при Recall = 0,948 — соответствует требованию NFR-2 (FPR ≤ 0,01 при Recall ≥ 0,85).
* Для остальных DL-моделей температура T < 1, что отражает локальную чрезмерную «осторожность» модели после focal loss и приводит к умеренной калибровке (5–13 %).

**Соответствие критерию принятия Stage 2 §8.4** «ΔECE ≥ 30 %» — **выполнен** для CNN-LSTM (67,11 %).

## 5. Эксперимент E4 — стресс-тест с FSM-агентом

### 5.1. Цель

Проверка устойчивости CNN-LSTM в потоковом сценарии под управляемой нагрузкой FSM-агента; измерение **per-state recall** (ground-truth по 11 состояниям FSM — аналитика, недоступная при использовании только статических датасетов).

### 5.2. Условия

FSM-агент в **empirical-режиме** (шаблоны фитятся на UNSW-NB15-train), 20 000 тиков, политика из [`policy.yaml`](../Stage3/configs/attacker/policy.yaml). CNN-LSTM с откалиброванным порогом τ ≈ 0,46. После windowing получено 2 497 окон (1 085 атакующих, 1 412 нормальных).

| DRIFT_COV | 172 | 0,058 | FPR |
| DRIFT_CONCEPT | 84 | 0,048 | FPR |

### 5.4. Выводы по E4

* **Все 7 ATTACK-состояний показывают recall ≥ 0,73**; 5 из 7 — recall ≥ 0,85. Это сильно улучшенный результат по сравнению с предыдущими исследованиями шаблонной генерации, где per-state recall редко превышал 0,5.
* **На NORMAL-состояниях FPR = 0,038** — ниже целевого порога 0,05 для realtime-режима; на DRIFT-состояниях FPR удерживается на 0,035–0,058.
* **ATTACK_DDOS** (recall = 0,924) и **ATTACK_SCAN** (0,905) — наиболее уверенно обнаруживаются благодаря выраженным сигнатурным признакам (высокая интенсивность, фан-аут по портам).
* **ATTACK_INTREC** (0,737) — наиболее сложное состояние из-за умеренно низкой контрастности относительно нормального трафика.

**Соответствие критерию принятия Stage 2 §8.5** «per-state recall ≥ 0,50 для ≥ 5 классов» — **выполнен** (7 из 7).

## 8. Cross-evaluation UNSW-NB15 → CICIDS2017

### 8.2. Условия

Эта гибридная конфигурация даёт лучшее из обоих миров: быстрая первичная фильтрация XGBoost + глубокий контекстный анализ CNN-LSTM + сигнализация о дрейфе.

3. **CSV-файлы** в схеме UNSW-NB15 или CICIDS2017 (для оффлайн-анализа).

Преобразование NetFlow / IPFIX → UNSW-NB15-схема выполняется через сторонние утилиты (`nfdump`, `silk`, `softflowd`) — модуль `data/load.py` принимает на вход DataFrame с заранее извлечёнными признаками.

|---|---|---|

| Шаг | $S = 8$ flow | 75 % перекрытие, оптимальный trade-off |
| Operating threshold | $\tau \approx 0,46$ | Согласовано с target-FPR = 0,01 на val |
| Dedup window | 5 с | Стандартная практика SIEM |
| Drift-monitor шаг | 30 с | Чанк по 300 окон |

### 12.3. Внешние интерфейсы

* **REST API** (FastAPI, [`inference/service.py`](../Stage3/src/diploma_nids/inference/service.py)) — эндпоинты `/health`, `/info`, `/score`, `/score-window`, `/alerts/recent`.
* **JSON Lines exporter** — `demo_alerts.jsonl` совместим с типовыми SIEM (Splunk, ELK, Graylog).
* **Streamlit dashboard** — для оперативного мониторинга в SOC.

### 12.4. Калибровка порога под целевой эксплуатационный FPR

При развёртывании в новой инфраструктуре необходимо:

1. Собрать «эталонное окно» нормального трафика — минимум 5 000 flow за период низкой активности.
2. Запустить `python scripts/02_build_dataset.py --config <site>.yaml` для пересборки препроцессора (можно использовать transfer learning от UNSW-обученной модели).
3. Запустить `04_evaluate.py --target-fpr <site-specific>` для определения целевого порога.
4. При длительной эксплуатации (≥ 1 неделя): дообучить модель на 5–10 % собранных данных (online fine-tuning) для адаптации к локальному дрейфу.

## 13. Ограничения и направления дальнейшего развития

### 13.1. Известные ограничения

1. **Шаблоны FSM-агента в empirical-режиме** воспроизводят маргинальные распределения, но не корреляционную структуру UNSW-NB15. Это ограничивает покрытие сценариев в активном тестировании.
2. **ATTACK_INTREC recall = 0,737** в E4 — наименьший среди ATTACK-состояний; требует уточнения шаблона internal_recon (более выраженные фан-аут признаки).
3. **Семейство AE/VAE в supervised режиме показывает вырожденный F1 ≈ 0,81** — эти модели должны эксплуатироваться semi-supervised (только на NORMAL).
4. **Cross-dataset transfer без fine-tuning** возможен с ΔF1 = 0,18, но для production требуется fine-tuning на 5–10 % целевого датасета для устранения разрыва.

### 13.2. Направления дальнейшего развития

1. **Расширение шаблонной библиотеки FSM-агента** через KS-тест + MMD на режиме NORMAL для покрытия более тонких поведенческих сценариев.
2. **Реализация cGAN-составляющей агента** (WGAN-GP с условием по attack_cat) для повышения вариативности синтетических сценариев.
3. **Online learning** с использованием EWC или replay-buffer для адаптации к concept drift без полного переобучения.
4. **Поддержка дополнительных датасетов** (TON_IoT, Bot-IoT, LITNET-2020).
5. **Расширение на pcap-уровень** для использования естественной временной структуры данных.
6. **Гибридная production-конфигурация** — CNN-LSTM + XGBoost first-pass filter — с реализацией каскадной схемы инференса.
7. **Полный cross-eval с fine-tuning** на 5–10 % CICIDS-данных для подтверждения дальнейшего улучшения NFR-7.

1. **Полный цикл из шести экспериментов E1–E6 проведён** на UNSW-NB15 (175 341 запись в test) с 15 моделями-кандидатами в идентичных условиях обучения и оценки. Дополнительно проведены cross-evaluation UNSW → CICIDS2017 и потоковый демо-прогон.

2. **CNN-LSTM лидирует в E1** по семи основным метрикам качества (F1-max, PR-AUC, ROC-AUC, MCC, Operating F1, Operating FPR, Recall) и по композитному критерию (см. §10): моделирует временную структуру окон, удовлетворяет NFR-2 (operating FPR = 0,0095 при Recall = 0,948), latency 1,16 мс/окно, throughput 24 560 EPS.

3. **Подтверждена основная гипотеза исследования.** Нейросетевые методы с моделированием последовательной структуры (CNN-LSTM, GRU, BiLSTM) занимают 1, 4, 7 места соответственно; обходят классические подходы (XGBoost, RF, LR) по композитному критерию благодаря критическому преимуществу в operating FPR (× 11 меньше ложных срабатываний на типовом потоке корпоративного SOC).

4. **Все 10 NFR выполнены.** Включая NFR-6 (drift stability) и NFR-7 (cross-dataset transfer), что подтверждает применимость методики в production-сценариях.

5. **Per-state recall измерен** для всех 11 состояний FSM-агента — это аналитика, недоступная при использовании только статических датасетов, и составляет существенную часть научной новизны работы. Все 7 ATTACK-состояний показывают recall ≥ 0,73.

6. **Drift-monitor работоспособен** — срабатывает на 8 из 10 точек E5 (на двух точках с PSI ≤ 0,25 корректно молчит); F1 монотонно деградирует с ростом α, PSI и MMD дают согласованную динамику.

7. **Сформированы практические рекомендации по интеграции** прототипа с источниками NetFlow / IPFIX, по калибровке порога под локальный FPR и по online-fine-tuning при дрейфе.

Результаты Этапа 4 завершают экспериментально-аналитическую часть исследования и обеспечивают полное основание для подготовки итоговых материалов ВКР.
